About

The tool

Neko Optimizer is a planning tool for The Battle Cats gacha. Give it a set of target cats and a budget of tickets and catfood, and it works out where and when to pull, in which order, for the lowest resource cost.

Every banner is driven by a single shared XOR-shift PRNG seed, so the whole future pull sequence is deterministic once the seed is known. Neko Optimizer rolls that sequence locally with its own engine, treats the pulls as a state graph, and searches it with A* and beam search for the cheapest path that collects your wishlist.

Who made it

Built by Iliyan Gavrilov as a personal project — an excuse to dig into pathfinding, deterministic RNG, and a full Django app end to end. The source lives on GitHub ↗.

Built with

  • Python 3.14 for the pure engine — RNG, roll engine, graph builder, and the A*/beam/subset search.
  • Django 6.0 for the web app, ORM, forms, and templates.
  • pytest and ruff for the test suite and linting, wired into CI.
  • File-based SQLite and committed JSON data — no server, runs fully offline.

Credits

The reverse-engineered gacha mechanics — the XOR-shift RNG and the roll-engine internals (dupe rerolls, bounces, guaranteed multis, track interleaving) — come from godfat's Battle Cats seed tracker ↗ (Apache-2.0). Neko Optimizer ports them to Python and validates the port byte-for-byte against godfat's output with a golden test; the planning layer on top is its own work.

Unit data and icons come from the battlecatsinfo ↗ catalogue, and the tier ranking from The Battle Cats Tier List ↗. Neko Optimizer is a fan-made tool and is not affiliated with PONOS.
